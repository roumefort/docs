# Contribution Guide

Thank you for your interest in contributing to our project! This guide will help you understand how to participate effectively.

---

## Prerequisites

Before you begin, make sure you have:

1. A GitHub account.
2. Git installed on your machine.
3. The project's dependencies properly set up (refer to the `README.md` file for instructions).

---

## Contribution Process

### 1. Fork the Repository

Click the **Fork** button at the top right of the repository page to create a copy in your own GitHub account.

### 2. Clone Your Fork

Clone your fork to your local machine:

```bash
git clone https://github.com/EpitechPromo2027/B-CPP-500-PAR-5-2-rtype-mikael.sourati R-TYPE
cd R-TYPE
```

### 3. Create a New Branch

Create a branch for your changes:

```bash
git checkout -b your-branch-name
```

**Branch Name :** Use a descriptive name, such as `feature/add-functionalities` or `bugfix/fix-bug`.

### 4. Make Your Changes

Make your changes using your favorite code editor. Be sure to follow the project's coding and documentation standards.

- **Commits :** Write clear and descriptive commits using 'Conventional Commits':

```bash
git add .
```

Name your commits following the 'Conventional Commits' standard.

Before pushing your changes, run the tests to ensure everything works as expected.

### 5. Push Your Changes

```bash
git push origin your-branch-name
```

---

### 6. Create a Pull Request

1. Navigate to the original repository.
2. Click on New Pull Request.
3. Select your branch and describe your changes.
4. Submit the Pull Request.

We will review your Pull Request as soon as possible and provide feedback if needed.

---

## Rules to Follow

- **Testing :** Any new feature or bug fix must be accompanied by tests.
- **Clean Code :** Follow coding standards and use comments where necessary.
- **Documentation :** Add or update documentation to reflect your changes.

---

## Assistance

If you have questions or need assistance, open an issue on GitHub or contact a project maintainer.
